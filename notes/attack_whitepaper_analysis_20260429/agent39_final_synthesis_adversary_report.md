## Cross-agent consensus

1. The stable paper is a normalized automorphic / Borcherds target paper first,
   not a compact Hall-source construction paper. Agents 01, 03, 10, 21, 23,
   and the attack PDF converge on this spine: virtual determinant, Gritsenko-
   Nikulin denominator target, OP/DT scalar square, D6-D2-D0 Gram dictionary,
   normal-ordered charge descent, and then an explicitly open compact
   realization problem.

2. The K3 input is a Grothendieck-group / protected-index package only.
   Agents 01, 03, 04, 13, 19, and 24 agree that `Z_K3 = 2 phi_{0,1}` and the
   Borcherds input `phi_{0,1}` must be described as an arithmetic half in
   `K0(sVect)`, not as a canonical Hilbert-space half, BPS half, parity split,
   or one-particle state space.

3. The normalized target identities are stable if stated with exact
   conventions. Agents 01, 03, 04, 11, 12, 20, 23, and 34 converge on:
   `D_X = Delta_5`, `D_5 = 64^{-1} Delta_5`, the BKM denominator
   `den(g_Delta5) = D_5(2Z) = 64^{-1} Delta_5(2Z)`, and OP/DT scalar branch
   `Z_OP/DT = -4096 Delta_5^{-2}`. The constants `64`, `4096`, and the OP
   minus sign are scalar-normalization data, not orientation data.

4. The Gritsenko-Nikulin denominator theorem imports a target algebra with
   signed supermultiplicities, not a microscopic compact source. Agents 01, 03,
   08, 23, 27, and 33 agree that the product determines signed coefficients on
   active support. It does not determine brackets, PBW, parity lifts, radicals,
   zero-superdimension summands, Hall constants, or compact primitive
   representatives.

5. The D6-D2-D0 dictionary is one of the paper's strongest new local
   contributions. Agents 02, 14, 17, 23, and 24 agree the repaired Mukai-Gram
   formula should be theorem-level: for the ideal sheaf convention,
   `Pi(Q_Y,P_Y) = (beta^2/2, n, d-1)`, and for `beta_h^2 = 2h-2` this is
   `(h-1,n,d-1)`. The ledger should record this as constructed, not as a
   lemma buried under source-side speculation.

6. The original fatal charge conflation is healed only if notation remains
   disciplined. Agents 01, 02, 14, 17, and 24 agree that the additive upstairs
   lattice is formal/Mukai, while the Fourier/root/Gram data live in
   `Gamma_gram = Z^3`; the map `Pi` is quadratic/non-additive. New prose should
   avoid bare `Gamma_X` and avoid `Gamma_X^phys` unless explicitly marked as
   mnemonic. The safer name is `Gamma_X^form`, with actual support
   `Gamma_{X,sigma}^{eff}` supplied separately.

7. The normal-ordered extension is a real theorem candidate; raw descent is a
   real no-go. Agents 02, 12, 17, 22, 23, and 27 converge on the need to
   distinguish the raw placement `i_0(c) = (c,0)` from the split section
   `s(c) = (c,Pi(c))`, define raw homogeneous / fixed-lift pushforwards before
   attacking them, and state the flag formula. The obstruction is not a
   nonzero ordinary cohomology class `[B]`; with `B = -delta Pi`, ordinary
   cohomology is zero. The obstruction is absence of an additive linear
   trivialization compatible with the required grading and source structures.

8. The `29|93` calculation is verified target arithmetic, not compact source
   evidence. Agents 08, 12, 25, and 27 agree that the first timelike split is
   useful as a finite target test, but it proves no compact primitive
   representatives, no bracket matrices, no radical quotient, no no-extra-
   relations theorem, and no PBW theorem.

9. The compact Hall/factorization source remains unconstructed. Agents 06, 08,
   16, 21, 25, 29, and 33 agree that any claim of compact source realization
   must supply finite substacks, correspondences, six-functor / vanishing-cycle
   data, orientation-gerbe data, primitive representatives, pairing/radical
   data, relations, PBW, and transition maps. Current target formulas do not
   supply these objects.

10. Target/source firewall is non-negotiable. Agents 15, 28, and 33 agree that
    a formal current envelope, target bar-cobar object, or algebraic regrading
    of the denominator algebra can be a useful target package. None of these
    is a compact source coalgebra, a Hall category, or an `E_3` realization.

11. Orientation must be gerbe-first. Agents 05, 07, 29, 31, and 34 agree that
    Maass characters, OP scalar signs, and scalar square roots do not produce
    Hall orientations. The hierarchy is `Hpre -> Or -> Htw -> Hor -> epsilon_o`
    only after explicit conditions: square-root gerbes, lifted
    correspondences, stabilizer/quotient checks, wall/Weyl cocycles, and
    orientation trivializations.

12. The eight Clery-Gritsenko rows and the spin L-factor are not core spine
    dependencies. Agents 09, 21, 23, and 31 agree that the row material can be
    an independent denominator-certificate ledger or companion note. The spin
    L-factor belongs in a companion arithmetic normalization note or a short
    non-dependent pointer, not in the main Dirac-Igusa proof chain.

13. Primary-source and build hygiene are still ledger items. Agents 11, 20, and
    26 agree that current citations resolve, but exact theorem-role anchors
    remain needed for GN/GNII, OP/Oberdieck/Bryan, group cohomology, row
    products, and orientation/factorization conventions. Semantic label drift
    and severe overfull boxes remain build risks after any source rewrite.

## Contradictions/tensions

1. Minimal stable paper versus construction-first paper. Agent 10 and the
   later attack-PDF synthesis outline a maximal construction-first architecture
   with `D^{alg}`, `Hpre`, `Htw`, hybrid sources, source coalgebras, and compact
   comparison. Agents 16, 21, 22, 23, 25, 26, 28, 29, and 33 warn that these
   objects are not currently built at source level. Integrated decision:
   ledger the minimal stable theorem spine as the current paper. Allow
   `D^{alg}` / lifted target regrading only if inserted as target-only algebra.
   Do not mark `Hpre`, `Htw`, hybrid source, source coalgebra, or compact
   comparison as constructed.

2. `Hpre` existence is contested by wording, not by desire. The attack PDF and
   Agent 16 describe a possible pre-Hall correspondence object. Agents 06, 25,
   29, and 33 show the missing proof obligations: finite windows, derived
   Artin geometry, exact-triangle correspondences, properness/support, residual
   inertia, vanishing cycles, and compatibility with normal ordering. Ledger
   decision: `Hpre` is a construction program or conditional interface, not a
   theorem, unless the manuscript supplies those stacks and correspondences.

3. Orientation gerbe versus twisted Hall object. Agents 07 and 29 suggest a
   gerbe-first repair is plausible. Agents 05, 06, 25, and 34 show that a
   gerbe is not yet a protected Hall algebra. Ledger decision: square-root
   gerbe definitions can be local/appendix material; `Htw` requires new
   source-side mathematics and cannot be inferred from scalar constants,
   Maass multipliers, or OP signs.

4. Algebraic target lift versus compact source. Agents 15 and 28 say a
   curve-universal regraded denominator target with a noncanonical lift `L` and
   section `s_L` is safe if target-only. Agent 22 warns not to add `Dalg` unless
   the objects are actually in the paper. Ledger decision: a target-only
   `D^{alg}_{Delta5,C,L}` may be status "constructible formal target" if
   defined. It must carry a forbidden-inference cell: not canonical, not
   compact, not Hall, not Pfaffian, not source.

5. Rows are useful but structurally dangerous. Agent 09 gives the strongest
   defense of the eight-row ledger as a genuine denominator-certificate table.
   Agents 21, 23, and 31 argue it distracts from the N=1 spine and invites
   invalid CHL/physical-host claims. Ledger decision: rows can survive only as
   an independent row-certificate appendix or companion note with per-row
   provenance, not as evidence for the compact source theorem.

6. OP/DT scalar square placement is not a mathematical disagreement. Agents 04
   and 34 want it late and scalar-only. Agent 10's maximal architecture can
   place it after first-order structure as a shadow. Ledger decision: late
   scalar consistency theorem in the stable paper; never an orientation,
   Pfaffian, or Hall-source input.

7. Naming can reintroduce the original bug. The attack PDF sometimes uses
   compressed names like `Gamma_X`; Agent 24 warns that the current manuscript
   already has safer notation. Ledger decision: forbid new ambiguous aliases.
   Use `Gamma_X^form`, `Gamma_gram`, `Pi_X`, `widehat Gamma_X`,
   `overline Pi_X`, and explicit source support names.

8. Recognition theorem versus recognition certificate. Agent 08 allows the
   primitive-recognition section as a certificate/checklist. Agents 01, 23, 25,
   and 27 show it is tautological or false if stated as a construction from
   target data. Ledger decision: demote all recognition wording to certificate
   or open comparison until source matrices exist.

9. Source construction pressure versus no-overclaim discipline. Several reports
   correctly object that a paper with only caveats can become inert. That does
   not justify fabricating the compact source. The ledger must distinguish
   "can be patched now", "can be defined target-only", "conditional interface",
   and "requires new mathematics".

## What requires new mathematics

1. A compact finite source. The paper needs actual finite Hall/factorization
   data: finite HN windows, source substacks or categories, correspondences,
   six-functor formalism, vanishing cycles, residual inertia control, and
   transition maps. Target coefficients cannot substitute for this.

2. `Hpre`. A pre-Hall object needs constructed stacks/categories, labels,
   exact-triangle correspondences, associativity, support conditions, and
   compatibility with the normal-ordered grading. It is not obtained by naming
   `Hpre`.

3. `Htw`. A twisted Hall object needs determinant lines, square-root gerbes,
   lifted correspondences, quotient-stack integration over `[Or/E]`, a
   tautological anti-invariant coefficient system, product/coproduct
   compatibility, and transition data. OP signs and Maass characters do not
   supply any of this.

4. Hybrid local/wrapped source. The local `K3` direction and wrapped `E`
   direction need a real mixed source with LL/LW/WL/WW operations, finite-first
   filtration, quotient-after-correspondence discipline, and QME/anomaly
   cochains.

5. Source coalgebra and Koszul object. A source chiral coalgebra must come from
   source geometry. Target bar-cobar sanity checks are not source coalgebras.
   The comparison direction must be `Omega C_X -> Den`, not "define C_X by the
   target bar".

6. Primitive comparison. Required data include source representatives, parity
   splits, bracket/coproduct/pairing matrices, radical ideals/coideals,
   relation matrices, no-extra-relations kernels, PBW checks, and transition
   compatibility. The W<=3 target table is only the expected output.

7. Orientation completion. The O1/O1+/O2 package needs finite-stabilizer
   classes, Weyl/wall cocycles, monodromy checks, and a comparison between the
   Hall orientation character and the automorphic Maass character. Until then,
   orientation remains a gerbe-level or conditional object.

8. Pfaffian-to-automorphic comparison. A compact Dirac/Pfaffian theorem needs a
   skew block, a Pfaffian line, a square map, a section/trivialization, and an
   automorphic line comparison `iota_aut`. The current determinant identity is
   a superdeterminant identity, not a Pfaffian theorem.

9. OP scalar trace from source. The imported scalar theorem is stable. Deriving
   it as a trace/shadow of the compact Dirac object requires the compact source
   and orientation package first.

10. Exact primary-source anchoring for high-stakes imports. GN/GNII, OP/
    Oberdieck/Bryan, group cohomology, row products, and factorization
    conventions need theorem-level citations, not only bibliography presence.
